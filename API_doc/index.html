Equb API Documentation

Base URL: https://equbapi.nehabi.com/api/v1

1. Get All Settings

Endpoint: GET /setting

Description: Retrieves a paginated list of settings.

Query Parameters:

Parameter Type Description
_page optional Specifies the page number for pagination.
_limit optional Specifies the number of results per page.
Example Request: 
GET https://equbapi.nehabi.com/api/v1/setting?_page=1&_limit=10
Response: 
[
    {
        "id": "abc123",
        "name": "notificationTime",
        "numericValue": 20
    },
    {
        "id": "def456",
        "name": "maxUsers",
        "numericValue": 50
    }
]

2. Get Setting by ID

Endpoint: GET /setting/:id

Description: Retrieves a specific setting by its ID.

Path Parameter:

  • :id: The ID of the setting to retrieve.
Example Request: 
GET https://equbapi.nehabi.com/api/v1/setting/abc123
Response: 
{
    "id": "abc123",
    "name": "notificationTime",
    "numericValue": 20
}

3. Add New Setting

Endpoint: POST /setting

Description: Creates a new setting.

Request Body:

{
    "name": "notificationTime",
    "numericValue": 20
}
Example Request: 
POST https://equbapi.nehabi.com/api/v1/setting
Content-Type: application/json

{
    "name": "notificationTime",
    "numericValue": 20
}
Response: 
{
    "id": "ghi789",
    "name": "notificationTime",
    "numericValue": 20
}

4. Update Setting by ID

Endpoint: PATCH /setting/:id

Description: Updates an existing setting by its ID.

Path Parameter:

  • :id: The ID of the setting to update.

Request Body:

{
    "name": "notificationTime",
    "numericValue": 25
}
Example Request: 
PATCH https://equbapi.nehabi.com/api/v1/setting/abc123
Content-Type: application/json

{
    "name": "notificationTime",
    "numericValue": 25
}
Response: 
{
    "id": "abc123",
    "name": "notificationTime",
    "numericValue": 25
}

5. Get All Setting Types

Endpoint: GET /setting-types

Description: Retrieves a paginated list of setting types.

Query Parameters:

Parameter Type Description
_page optional Specifies the page number for pagination.
_limit optional Specifies the number of results per page.
Example Request: 
GET https://equbapi.nehabi.com/api/v1/setting-types?_page=1&_limit=10
Response: 
[
    {
        "id": "type1",
        "name": "Integer"
    },
    {
        "id": "type2",
        "name": "String"
    }
]
